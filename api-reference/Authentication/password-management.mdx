# Forgot Password (Send Reset Link)
**POST** <Badge>/api/auth/forgot-password</Badge>

Sends a password reset link to the user's registered email address.

**Headers**
| Name                                   | Value      |
| --------------------------------------- | ---------  |
|  Content-Type              | <Badge>application/json</Badge> |

**Body :**
```
{
  "email": "student@example.com"
}
```

**Validation Rules:**
| Field                                   | Type      | Required  | Description
| --------------------------------------- | --------- | --------- | ------------------------------------  |
|  <Badge>email</Badge>              |  string  | ✅ Yes    | Must be a valid registered email address. |


**Response :**
<Tabs>
  <Tab title="200">
    ✅ 200 OK -  Success
      ```
{
  "message": "Reset password link sent on your email."
}
    ```
  </Tab>
  <Tab title="500">
    ❌ 500 Internal Server Error

 ```
{
  "message": "An error occurred, please try again later."
}
 ```
  </Tab>
</Tabs>

# Reset Password

Resets the user’s password using the token sent to their email.

The token is included in the password reset link.

**POST**<Badge>/api/auth/reset-password</Badge>

**Headers**
| Name                                   | Value      |
| --------------------------------------- | ---------  |
|  Content-Type              | <Badge>application/json</Badge> |

**Body :**
```
{
  "token": "example-reset-token",
  "email": "student@example.com",
  "password": "newSecurePassword",
  "password_confirmation": "newSecurePassword"
}
```

**Validation Rules:**
| Field                                   | Type      | Required  | Description
| --------------------------------------- | --------- | --------- | ------------------------------------  |
|<Badge>token</Badge> |string|✅ Yes|The token provided in the reset link|
|  <Badge>email</Badge>              |  string  | ✅ Yes    | The same email that requested the reset |
|<Badge>passowrd</Badge> |string|✅ Yes|New password (min: 3, max: 255)|
|<Badge>password_confirmation</Badge> |string|✅ Yes|Must match the new password|


**Response :**
<Tabs>
  <Tab title="200">
    ✅ 200 OK -  Success
      ```
{
  "message": "Your password has been successfully reset."
}
    ```
  </Tab>
  <Tab title="500">
    ❌ 500 Internal Server Error
 ```
{
  "message": "An error occurred, please try again later."
}
 ```
  </Tab>
</Tabs>

# Update/Change Password (Authenticated)

Allows a logged-in user to update their password by providing their current password and the new one.

**POST** <Badge>/api/auth/update-password</Badge>

**Headers**
| Name                                   | Value      |
| --------------------------------------- | ---------  |
|  Content-Type              | <Badge>application/json</Badge> |
|Authorization|<Badge>Bearer `<token>`</Badge> |

**Body :**
```
{
  "current_password": "oldPassword123",
  "new_password": "newSecurePassword456",
  "new_password_confirmation": "newSecurePassword456"
}
```

**Validation Rules:**
| Field                                   | Type      | Required  | Description
| --------------------------------------- | --------- | --------- | ------------------------------------  |
|  <Badge>current_password</Badge>              |  string  | ✅ Yes    | The user’s current password |
|<Badge>new_password</Badge> |string|✅ Yes|New password (min: 3, max: 255)|
|<Badge>new_password_confirmation</Badge> |string|✅ Yes|Must match the new password|


**Response :**
<Tabs>
  <Tab title="200">
    ✅ 200 OK -  Success
      ```
{
    "message": "Your password has been successfully reset."
}
    ```
  </Tab>
  <Tab title="401">
    ⚠️ 401 Unauthorized

 ```
{
  "message": "Please verify your old password."
}
 ```
  </Tab>
  <Tab title="500">
    ❌ 500 Internal Server Error

 ```
{
  "message": "An unexpected error occurred."
}
 ```
  </Tab>
</Tabs>

**Notes**
- The **Forgot Password** API sends an email to the user with a password reset link.
The frontend should display a success message even if the email doesn’t exist (to avoid information leaks).

- The **Reset Password** API expects the token and email parameters from the reset link in the email.
After a successful reset, redirect the user to the login page.

- The **Update Password** API requires the user to be logged in (use the same access token as for other authenticated requests).
After a successful update, you can prompt the user to log in again for security.
